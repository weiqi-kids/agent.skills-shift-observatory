# 如何使用本站

第一次來嗎？這個頁面幫助你快速上手。

---

## 第一步：選擇你的角色

不同角色有不同的閱讀需求，我們為你準備了專屬入口：

| 你是... | 建議入口 |
|:--------|:---------|
| 正在找工作或考慮轉職 | [求職者專區](/for-job-seekers/) |
| 有工作，想保持競爭力 | [在職者專區](/for-professionals/) |
| HR 或企業主管 | [HR/企業專區](/for-employers/) |
| 研究者或政策制定者 | [研究者專區](/for-researchers/) |

每個專區都有針對該角色的推薦報告和閱讀順序。

---

## 第二步：了解五大報告

本站每週產出 5 份報告，各有不同用途：

| 報告 | 回答什麼問題 | 適合誰 |
|:-----|:-------------|:-------|
| [景氣溫度計](/climate-index/) | 市場好嗎？哪些產業在擴張？ | 所有人 |
| [技能漂移](/skills-drift/) | 哪些技能正在變熱門？ | 求職者、在職者 |
| [產業分層](/industry-segments/) | 各產業的用人方向？ | HR、投資人 |
| [薪資帶](/salary-bands/) | 合理薪資期望是多少？ | 求職者、HR |
| [求職策略](/career-strategy/) | 我該怎麼規劃職涯？ | 求職者 |

### 建議閱讀順序

如果你想全面了解市場，建議這個順序：

```
景氣溫度計 → 技能漂移 → 產業分層 → 薪資帶 → 求職策略
```

這個順序是從「大局」到「細節」，從「觀測」到「行動」。

---

## 第三步：學會看報告

### 理解術語

報告中有一些專業術語，建議先了解：

- **P25 / P50 / P75**：薪資分布的統計方式。[什麼意思？](/glossary/#p25--p50--p75)
- **AI 取代向量**：分析不同工作類型的自動化風險。[什麼意思？](/glossary/#ai-取代向量)
- **溫度等級**：用溫度來形容市場狀態。[什麼意思？](/glossary/#景氣溫度)

完整術語表請見 [名詞解釋](/glossary/)。

### 注意報告結構

每份報告都有類似的結構：

1. **摘要**：2-3 句話總結核心發現
2. **核心數據**：表格呈現的關鍵指標
3. **分析內容**：詳細的趨勢分析
4. **免責聲明**：使用限制說明

建議先看摘要，再決定要不要深入閱讀。

### 注意免責聲明

每份報告底部都有免責聲明。**請務必閱讀。**

簡單說：這些報告是「市場觀測」，不是「個人建議」。重大決策前請諮詢專業人士。

## 搜尋功能

右上角有搜尋框，可以直接搜尋：

- 職業名稱（如「軟體工程師」）
- 技能名稱（如「Python」）
- 產業名稱（如「半導體」）

搜尋結果會顯示相關報告的片段。

---

## 查看歷史報告

每個報告類型都保留了歷史週次：

1. 點擊左側導覽列的報告名稱（如「景氣溫度計」）
2. 展開後可以看到各週次（W06、W07、W08...）
3. 點擊週次進入該週報告

這讓你可以比較不同週次的變化趨勢。

## 常見問題

### 報告看不懂怎麼辦？

先看 [名詞解釋](/glossary/)，了解專業術語。如果還有問題，歡迎透過 [GitHub Issues](https://github.com/weiqi-kids/agent.skills-shift-observatory/issues) 提問。

### 我想要更詳細的資料？

每份報告都標註了資料來源。如果你是研究者，可以直接到原始來源取得更詳細的資料。詳見 [研究者專區](/for-researchers/)。

### 報告有錯誤？

請透過 [GitHub Issues](https://github.com/weiqi-kids/agent.skills-shift-observatory/issues) 回報，我們會盡快處理。
