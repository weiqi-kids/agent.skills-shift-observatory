# Mode：skills_drift / 技能需求漂移分析

## Mode 定義表

| 項目 | 說明 |
|------|------|
| **Mode name** | skills_drift / 技能需求漂移分析 |
| **Purpose and audience** | 追蹤技能標籤在職缺中的出現頻率變化，供技術工作者、教育機構、HR 了解技能需求趨勢 |
| **Source layers** | tw_104_jobs（主要）、tw_govjobs、global_stackoverflow、global_linkedin_workforce、global_wef_jobs |
| **Automation ratio** | 70% — 頻率統計可自動化，趨勢解讀需 AI 綜合分析 |
| **Content risk** | 技能標籤粒度不一（"Python" vs "Python 資料分析"），可能產生誤導性排名 |
| **Reviewer persona** | 幻覺風險審核員 + 領域保守審核員 |

## 資料來源定義

讀取以下 Layer 最近一個觀測週期的萃取結果（`docs/Extractor/{layer_name}/` 下的 `.md` 檔）：

| Layer | 讀取重點 | 優先度 |
|-------|----------|--------|
| tw_104_jobs | 職缺中的技能標籤、出現頻率、各產業/角色的技能需求 | 主要來源 |
| tw_govjobs | 政府職缺技能需求、公部門數位轉型技能趨勢 | 補充來源 |
| global_stackoverflow | 全球技術技能趨勢、開發者工具偏好變化 | 交叉驗證 |
| global_linkedin_workforce | 全球技能需求趨勢、新興技能標籤 | 交叉驗證 |
| global_wef_jobs | 世界經濟論壇未來技能預測、長期趨勢 | 趨勢背景 |

### 技能標籤處理原則

1. **標準化**：將同義技能合併（例如 "JS" = "JavaScript"、"ML" = "Machine Learning" = "機器學習"）
2. **粒度控制**：優先使用中粒度標籤（例如 "Python" 而非 "Python 3.11"，但區分 "Python 資料分析" 與 "Python Web 開發"）
3. **分類歸屬**：每個技能標籤歸入以下分類之一：
   - 程式語言（Programming Languages）
   - 框架與工具（Frameworks & Tools）
   - 雲端與基礎設施（Cloud & Infrastructure）
   - 數據與 AI（Data & AI）
   - 資安（Security）
   - 軟技能（Soft Skills）
   - 領域知識（Domain Knowledge）
   - 證照（Certifications）

## 五大 AI 取代向量對應的技能觀察維度

| 取代向量 | 觀察重點 | 代表性技能類別 |
|----------|----------|---------------|
| 認知例行（cognitive_routine） | 資料輸入、記帳、報表等技能需求是否下降 | Excel 基礎、ERP 操作、基礎會計軟體 |
| 認知非例行（cognitive_nonroutine） | 分析、設計、策略等高階認知技能需求變化 | 資料分析、UX 設計、策略規劃、AI/ML |
| 體力例行（physical_routine） | 製造、倉儲等體力例行技能的自動化替代 | 產線操作、倉儲管理、品管檢測 |
| 體力非例行（physical_nonroutine） | 需靈活應變的體力工作技能 | 設備維修、現場施工、醫療照護操作 |
| 高度人際（interpersonal） | 溝通、管理、銷售等人際技能需求 | 領導力、客戶管理、簡報能力、跨部門協作 |

## 執行注意事項

1. **標籤合併規則**：合併同義標籤時，使用最通用的名稱，並在首次出現時以括號標註別名（例如「JavaScript（JS）」）。
2. **小樣本警告**：當某技能的出現次數低於 5 次時，變化率數據不可靠，應標註「小樣本」。
3. **季節性排除**：特定季節性技能（如「報稅軟體」在報稅季）的變化不應被誤判為趨勢。
4. **數據不足降級**：若主要來源（tw_104_jobs）無新數據，本週報告應降級為「僅全球趨勢摘要」，並明確標註。
5. **歷史累積**：本 Mode 需要至少 4 週的歷史數據才能產出完整報告。首次執行時，僅產出「基線建立」版本。
