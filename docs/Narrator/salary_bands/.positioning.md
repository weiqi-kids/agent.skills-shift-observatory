## 1. 核心資訊

| 項目 | 內容 |
|------|------|
| **Mode 名稱** | salary_bands / 薪資帶分析 |
| **一行描述** | 每週分析 53 個職業角色的 P25/P50/P75 薪資帶，讓薪資談判不再靠感覺，讓企業對標不再靠年度報告 |
| **核心價值主張** | 在台灣，薪資透明度長期偏低，「面議」文化讓求職者永遠處於資訊弱勢。薪資帶分析用每週更新的市場數據打破資訊不對稱——你應該拿多少、哪個產業給最多、AI 取代風險高的角色薪資是在漲還是在跌 |

### 價值主張細化

- **給求職者**：開口喊薪之前，先知道市場 P50 在哪裡，不要因為資訊不對等少拿了 10-20%
- **給 HR / 薪酬設計**：年度薪資調查報告太舊，用週頻的市場數據做薪資帶設計，保持競爭力
- **給職涯顧問 / 獵頭**：有數據支撐的薪資建議，比「市場行情大概是……」更有說服力

## 2. 目標受眾

### 主要受眾

#### 受眾 A：正在求職或準備跳槽的工作者

| 面向 | 說明 |
|------|------|
| **人口特徵** | 25-45 歲，準備換工作或接受 offer 評估，最關注「我應該要求多少薪資」 |
| **核心需求** | 知道自己目標職位的市場薪資水準（P25/P50/P75），有依據開薪資條件；知道跨產業薪資差異 |
| **痛點** | 「面議」職缺多、道聽塗說、Dcard 或 PTT 的薪資分享樣本小且不具代表性；缺乏系統性的台灣薪資數據 |
| **使用場景** | 收到 offer 前確認薪資合理性；決定是否接受現有薪資調整；準備薪資談判時建立心理錨點 |
| **行動觸發點** | 看到自己角色的 P75 數字，發現明顯高於現有薪資，增加談判信心 |
| **關鍵情緒** | 希望感（我可以拿更多）+ 安全感（我現在的薪資是正常的） |

#### 受眾 B：企業薪酬福利主管 / HRBP

| 面向 | 說明 |
|------|------|
| **人口特徵** | 薪酬福利專員/主管、HR Business Partner，負責薪資帶設計與市場對標 |
| **核心需求** | 即時的市場薪資數據，作為薪資帶校準和年度調薪的外部依據 |
| **痛點** | 年度薪資調查（Mercer、Hay Group）報告昂貴（數十萬元年費）；免費數據時效落後 1 年以上 |
| **使用場景** | 新職缺薪資帶設定、年度調薪幅度參考、候選人薪資談判時的外部數據支撐 |
| **差異化需求** | P25/P50/P75 分位數數據（而非平均值）；跨產業比較；角色到職缺的顆粒度對應 |

| **使用場景** | 研究引用、政策倡議數據支撐 |

## 3. 差異化定位

### 相對競爭者的差異

| 競爭者類型 | 代表 | 我們的差異 |
|------------|------|------------|
| **商業薪資調查** | Mercer、Hay Group、Korn Ferry 薪資報告 | 免費 vs 高額年費；週頻 vs 年度；53 角色全覆蓋 vs 特定產業聚焦 |
| **職缺平台薪資** | 104 薪資報告、1111 薪資指南、Glassdoor | 系統性多角色對比 vs 個別職缺瀏覽；P25/P50/P75 分位數 vs 平均值；AI 取代向量整合 |
| **社群薪資分享** | PTT Soft_Job 薪資板、Dcard 薪資文 | 代表性樣本 vs 自願揭露偏差；週頻更新 vs 隨機散佈；匿名系統性 vs 個人案例 |
| **政府薪資統計** | 行政院主計處職類薪資調查 | 週頻 vs 年度；職業角色顆粒度 vs 大類職業分類 |

### 核心獨特優勢

1. **P25/P50/P75 三分位呈現**：不只說「平均薪資」，用三分位讓薪資分布清晰，避免極端值誤導
2. **53 角色系統覆蓋**：從軟體工程師到護理師，跨越認知/體力/人際三大取代向量
3. **跨產業薪資差異**：同一職稱在不同產業的薪資差距呈現（如「軟體工程師在半導體 vs 教育業」）
4. **「面議」問題透明化**：明確說明面議職缺排除後的樣本偏差，讓讀者知道數據的侷限性
5. **全球對標**：台灣薪資 vs 美國 BLS、全球 Hays 數據，讓讀者判斷自己在全球薪資地圖的位置
6. **AI 取代向量薪資關係**：高 AI 取代風險的角色薪資是否真的在降？數據說話

### 我們不做的事

- 不保證薪資數據精確反映個人實際薪資（結構性局限性必須揭露）
- 不提供薪資談判話術或具體建議句式（超出數據報告範疇）
- 不對性別、年齡、種族薪資差距做個人歸因分析
- 不推薦特定公司或職位

## 4. 品牌調性

### 語氣 (Tone of Voice)

| 維度 | 描述 |
|------|------|
| **整體調性** | 財務顧問的精確 + 職涯教練的務實。「數字是這樣的，對你的意義是……」，具體且有方向感 |
| **對求職者說話** | 賦權語氣（你有資格知道市場價值）、去除神秘感（薪資不是不可說的秘密） |
| **對 HR 說話** | 專業薪酬術語（P25/P50/P75、薪資帶、外部競爭力），平等對話 |
| **對研究者說話** | 方法論透明，主動揭露數據局限性 |
| **應避免** | 「你應該拿 X 萬」的確定性建議；迴避面議問題導致的數據偏差；性別刻板印象角色薪資討論 |

### 視覺語言

| 元素 | 風格 |
|------|------|
| **薪資帶橫條圖** | P25-P75 的範圍橫條，P50 用垂直線標示，清晰呈現薪資分布 |
| **跨產業薪資熱力圖** | 53 角色 × 產業的薪資密度，讓高薪組合一眼可見 |
| **AI 取代向量薪資矩陣** | 5 個取代向量的薪資中位數比較，呈現薪資與取代風險的關係 |
| **全球對標箭頭** | 台灣 P50 vs 全球參考值的方向差距，視覺化薪資競爭力 |

### 關鍵詞

- 台灣薪資水準、薪資行情、薪資談判
- 軟體工程師薪資、{職位}月薪中位數
- 薪資帶分析、P50 薪資、薪資分位數
- 跨產業薪資比較、薪資透明度
- AI 取代薪資影響、高薪職業

## 6. 改版目標

### P0（阻擋發布的問題）

- [ ] 「面議排除聲明」必須出現在薪資數據的顯眼位置（不能埋在頁尾免責聲明）
- [ ] P25/P50/P75 三個分位數必須同時呈現，不能只給平均值或中位數
- [ ] 樣本數不足時（如某角色本週觀測職缺 < 10 筆）必須明確標示「樣本量不足，參考價值有限」

### P1（重要優化，下次迭代完成）

- [ ] 增加「薪資變化趨勢」：本週 P50 vs 4 週前 vs 12 週前的歷史軌跡
- [ ] 加入「跨產業薪資差異」：同一角色在 14 個產業的薪資比較熱力圖
- [ ] 增加「全球對標」：台灣 P50 vs 美國 BLS、Hays 全球參考的直觀對比
- [ ] 提供「薪資計算機」工具：讓讀者輸入職位和產業，即時查看自己的市場位置

### P2（長期優化）

- [ ] 建立薪資歷史資料庫（按季度），讓讀者追蹤特定職位的薪資長期走勢
- [ ] 開發薪資 API，讓 HR 系統可程式化整合市場薪資參考數據
- [ ] 與學術機構合作進行薪資數據方法論驗證，提升數據公信力
- [ ] 探索新增「地區薪資差異」維度（台北 vs 新竹 vs 台中 vs 高雄）
