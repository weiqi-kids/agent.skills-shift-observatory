## 1. 受眾深度分析

### 1.1 主要受眾 A：正在求職或準備跳槽的工作者

#### Persona 摘要

| 項目 | 描述 |
|------|------|
| 角色 | 準備換工作或評估 offer 的工作者 |
| 年齡/背景 | 25-45 歲，最關注「我應該要求多少薪資」 |
| 目標 | 知道自己目標職位的市場薪資水準，有依據開薪資條件 |
| 痛點 | 面議文化、PTT/Dcard 分享樣本小、缺乏系統性台灣薪資數據 |
| 行為特徵 | 收到 offer 時急切搜尋，會直接搜尋「{職位}薪資」 |

#### Jobs-to-be-Done

| 任務類型 | 具體任務 |
|----------|----------|
| 功能性 | 確認目標職位的 P25/P50/P75 薪資範圍，設定合理薪資期望 |
| 功能性 | 比較同職位在不同產業的薪資差異，選擇薪資最好的產業投遞 |
| 功能性 | 評估手上 offer 的薪資是否合理（在 P25 以下還是 P50 以上？） |
| 情感性 | 希望感——「我可能可以拿到更高的薪資」 |
| 情感性 | 安全感——「我現在的薪資在市場上是正常的」 |
| 情感性 | 降低薪資談判前的焦慮，有數據當心理錨點 |
| 社會性 | 在薪資談判中展現「我了解市場行情」的專業態度 |

#### 使用者旅程

| 階段 | 行為 | 問題/疑慮 | 需要的資訊 | 期望的體驗 |
|------|------|-----------|------------|------------|
| 意識 | 搜尋「軟體工程師薪資」「{職位}月薪」 | 有沒有可靠的台灣薪資數據？ | SEO 含職位名稱和薪資 | 搜尋結果直接顯示 P50 數字 |
| 考慮 | 找到自己的角色分類 | 這個數據可信嗎？面議的被排除了嗎？ | 面議排除說明、樣本量 | 立即看到數據品質聲明 |
| 決策 | 比較自己 offer vs 市場 P50 | 我在 P25 以下是不是被低估了？ | P25/P50/P75 三個數字 | 數字清晰、直觀可比較 |
| 使用 | 引用數據在薪資談判中 | 這個數據夠新嗎？ | 最近更新日期、觀測週期 | 可引用的格式 |
| 推薦 | 分享到 PTT/Dcard 求職板 | 這個報告每週都有嗎？ | 連續更新紀錄 | 社群分享友好 |

#### 資訊需求

| 想知道什麼 | 擔心什麼 | 需要的證據 |
|------------|----------|------------|
| 我的職位市場行情是多少？ | 數據只反映低薪職缺（高薪面議被排除） | 面議排除比例透明揭露 |
| 哪個產業給的薪資最高？ | 跨產業比較是否公平（同職位不同年資） | 樣本量、經驗年資說明 |
| 全球來看台灣薪資是什麼水準？ | 台灣薪資是不是真的很低 | 台灣 vs 美國 vs 全球的對比 |

## 5. 關鍵發現摘要

### 最重要的 3 個發現

1. **角色覆蓋率（16/53）是最大的信任風險**：首頁宣稱追蹤 53 個角色，但實際僅 30% 有有效薪資數據。這不只是數據缺口，更是期待管理的問題。必須在首頁和報告中明確說明現況。

2. **面議排除透明度是最大的差異化資產**：台灣薪資數據的核心問題是面議文化，本站公開揭露 47.1% 面議排除率，是業界罕見的透明度。應將此作為品牌核心信任標章來經營。

3. **薪資談判參考是求職者最迫切的需求但目前完全缺失**：報告提供了 P25/P50/P75 數據，但沒有告訴讀者「這些數字對你意味著什麼」。在免責聲明框架下加入薪資談判參考段落，可大幅提升報告的實用價值。

### 改版重點方向

1. **期待管理修正**：明確說明角色覆蓋率現況，建立合理預期
2. **視覺化強化**：Top 10 薪資帶 Mermaid 長條圖，讓數字直觀化
3. **實用價值提升**：加入薪資談判參考段落，從「數據報告」升級為「決策參考工具」
