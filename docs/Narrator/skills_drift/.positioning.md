## 1. 核心資訊

| 項目 | 內容 |
|------|------|
| **Mode 名稱** | skills_drift / 技能需求漂移分析 |
| **一行描述** | 每週追蹤 53 個職業角色的技能標籤出現頻率變化，幫助你在技能需求轉移之前就看到信號 |
| **核心價值主張** | 職場技能的市場需求每週都在漂移，技能需求漂移分析讓你第一時間看到哪些技能正在崛起、哪些正在被市場淘汰——早一季看到，就多一季準備時間 |

### 價值主張細化

- **給技術工作者**：你學的技能還值錢嗎？MCP 需求暴增的意義是什麼？每週數據說話，不靠道聽塗說
- **給教育機構**：課程設計不再憑感覺，用市場數據驗證教學內容的時效性
- **給 HR**：候選人說自己熟悉某技術，你知道這個技術現在市場需求是上升還是下滑嗎？

## 2. 目標受眾

### 主要受眾

#### 受眾 A：技術工作者（工程師、資料科學家、設計師）

| 面向 | 說明 |
|------|------|
| **人口特徵** | 25-40 歲技術專業人士，有 2-10 年經驗，持續面對技術棧選擇壓力 |
| **核心需求** | 確認自己學習投資（時間、金錢）的方向是對的；知道哪些技能正在「漂移」而尚未被大眾發現 |
| **痛點** | 技術社群噪音大（每季都有「必學技術」列表）、難以分辨真正的市場需求與社群炒作 |
| **使用場景** | 決定下季學什麼技術、評估是否要轉換技術棧、確認目前技能的市場相關性 |
| **行動觸發點** | 看到自己熟悉的技能出現在「下降榜」，或陌生技能出現在「上升榜 Top 3」 |
| **差異化需求** | 不只要排行榜，要理解「為什麼」這個技能在上升，以及在哪些產業 / 哪類角色中上升 |

#### 受眾 B：技職教育機構 / 線上學習平台 / 訓練機構

| 面向 | 說明 |
|------|------|
| **人口特徵** | 課程設計師、教育總監、職業培訓機構負責人 |
| **核心需求** | 課程內容的市場驗證依據，避免開設市場需求下滑的課程，及早佈局新興技能課程 |
| **痛點** | 課程開發周期 3-6 個月，等到數據明顯時市場已飽和；政府補助課程需要有數據支撐申請 |
| **使用場景** | 課程主題選定、教學大綱更新優先序、講師招募方向 |

#### 受眾 C：企業 HR / 人才發展部門

| 面向 | 說明 |
|------|------|
| **人口特徵** | HRBP、L&D（學習與發展）負責人、技術招募主管 |
| **核心需求** | 了解外部市場技能需求趨勢，對齊內部人才培育計劃；制定技術職評估標準 |
| **痛點** | 內部技能評估與外部市場脫節，員工培訓預算缺乏外部市場依據 |
| **使用場景** | 年度培訓計劃制定、技術職招募 JD（職位描述）撰寫、薪資對標討論 |

### 次要受眾

#### 受眾 D：職涯顧問 / 獵頭

| 面向 | 說明 |
|------|------|
| **核心需求** | 為客戶提供有數據支撐的技能建議，提升顧問建議的說服力 |
| **使用場景** | 候選人技能差距分析、職涯路徑建議 |

## 3. 差異化定位

### 相對競爭者的差異

| 競爭者類型 | 代表 | 我們的差異 |
|------------|------|------------|
| **技術社群排行** | Stack Overflow 年度調查、GitHub Octoverse | 週頻 vs 年度；求職市場需求 vs 開發者使用偏好（兩者有落差） |
| **職缺平台技能統計** | 104 技能熱搜、LinkedIn Top Skills | 跨平台多來源整合 vs 單平台數據；AI 取代向量交叉分析 |
| **諮詢報告** | WEF Future of Jobs、Coursera Job Skills Report | 台灣在地數據 vs 全球泛稱；週頻 vs 年度 |
| **媒體報導** | 各科技媒體「必學技術」清單 | 數據驅動 vs 編輯判斷；可追蹤變化趨勢 vs 靜態快照 |

### 核心獨特優勢

1. **漂移概念**：不只是排行榜，而是強調「正在移動中」的動態變化，4 週與 12 週趨勢對比
2. **AI 取代向量交叉**：每個技能都標注落在哪個 AI 取代向量，讓技能學習與職業安全性掛鉤
3. **產業分層**：同一個技能在不同產業的需求變化不同，「Python 在金融業上升」vs「Python 在教育業下降」
4. **台灣本地數據為主**：WEF、LinkedIn 說的是全球趨勢，我們說的是台灣職缺市場的真實需求
5. **新出現技能早期預警**：新技能標籤首次出現在職缺中時立即追蹤，不等到成為「熱門技術」才報導

### 我們不做的事

- 不推薦具體學習平台或課程（避免利益衝突）
- 不對技能的「好壞」做定性評判，只呈現市場需求數據
- 不預測特定技能何時會消失（無法精確預測）
- 不進行個人技能評估或技能測試

## 4. 品牌調性

### 語氣 (Tone of Voice)

| 維度 | 描述 |
|------|------|
| **整體調性** | 數據分析師的精準 + 同行先行者的實用感。「這週我從數據中看到了什麼」而非「這週市場發生了什麼大事」 |
| **對技術人員說話** | 直接、具體、有技術深度，不過度解釋基本概念 |
| **對教育機構說話** | 提供決策依據而非結論，數據呈現客觀，解讀可由讀者判斷 |
| **對 HR 說話** | 連結技能趨勢與招募決策的實務含意 |
| **應避免** | 誇大炒作（「這個技能你不學就落伍了！」）；不確定時給出確定性結論 |

### 視覺語言

| 元素 | 風格 |
|------|------|
| **核心視覺** | 技能排行榜（上升 ↑ / 下降 ↓ / 持平 — 箭頭符號），讓變化方向一目了然 |
| **趨勢圖** | 4 週與 12 週趨勢對比，時間軸清晰 |
| **技能分類色碼** | 不同技能類別（程式語言 / 框架 / 雲端 / AI 等）用色碼區分 |
| **AI 取代向量標籤** | 每個技能條目附帶取代向量標籤，增加分析深度 |

### 關鍵詞

- 技能需求趨勢、熱門技能、技能漂移
- 程式語言排行、AI 技能、雲端技能需求
- 職缺技能分析、台灣技術人才市場
- 技能上升下降、新興技能、技能替代

## 6. 改版目標

### P0（阻擋發布的問題）

- [ ] 技能排行榜必須同時呈現 4 週和 12 週趨勢（單週數據波動過大，容易誤讀）
- [ ] 每個技能標注 AI 取代向量分類，讓技能選擇與職涯安全性直接連結
- [ ] 新出現技能（首次進入觀測）必須特別標示，不與既有技能混排

### P1（重要優化，下次迭代完成）

- [ ] 增加「產業別技能需求」切面：同一技能在不同產業的需求差異
- [ ] 增加「角色別技能需求」：軟體工程師 vs 資料科學家的技能組合比較
- [ ] 加入技能需求的地理分布（台北 vs 新竹 vs 台中技能差異）
- [ ] 提供技能標籤的歷史走勢 API，讓教育機構可程式化查詢

### P2（長期優化）

- [ ] 建立「技能組合」分析：哪些技能組合（skills bundle）出現在最高薪職缺中
- [ ] 加入全球技能趨勢對比：台灣 vs 美國 vs 歐洲的技能需求差異
- [ ] 開發「技能雷達圖」個人化工具，讓讀者輸入自己的技能後看到市場定位
