## 1. 核心資訊

| 項目 | 內容 |
|------|------|
| **Mode 名稱** | career_strategy / 求職策略建議 |
| **一行描述** | 整合本週所有市場觀測數據，為求職者和轉職者提供有數據支撐的職涯行動參考方向 |
| **核心價值主張** | 大多數職涯建議要麼太泛用（「提升軟技能！」）、要麼太個人化（需要昂貴的一對一諮詢）。求職策略建議站在中間地帶——用本週真實市場數據，告訴你現在具體可以採取哪些行動，但不代替你做決定 |

### 價值主張細化

- **給正在求職的人**：現在應該投什麼角色、補什麼技能、避開什麼產業？本週數據說，不是我說
- **給在職考慮轉職的人**：現在是好時機嗎？我的技能在哪些方向有轉移價值？
- **給擔憂 AI 取代的人**：我的職業在哪個取代向量？現在市場是否真的在縮減這類職缺？

### 重要定位說明

本 Mode 是系統中風險等級最高的報告。所有輸出必須明確定位為「基於數據觀測的參考方向」，而非個人化建議。這個定位是差異化的核心，也是公信力的基礎。

## 2. 目標受眾

### 主要受眾

#### 受眾 A：積極求職中的工作者

| 面向 | 說明 |
|------|------|
| **人口特徵** | 25-40 歲，已開始投遞職缺或即將開始，希望提高求職成效 |
| **核心需求** | 知道本週哪些角色和產業的機會較多；了解自己技能的市場匹配度；得到具體的行動建議 |
| **痛點** | 求職建議通常是泛用的，不知道「現在」和「我的情況」；花時間投遞卻沒有回音；不確定該補什麼技能才有效 |
| **使用場景** | 每週一打開，快速掌握「這週我應該重點關注什麼」；評估是否調整求職方向 |
| **行動觸發點** | 看到自己目標角色出現在「本週機會窗口」清單，立即加強投遞力道 |
| **心理需求** | 方向感（我知道該怎麼做）+ 信心感（數據說可以，不只是感覺） |

#### 受眾 B：在職考慮轉職 / 職涯轉換點的工作者

| 面向 | 說明 |
|------|------|
| **人口特徵** | 30-45 歲，有一定工作經驗，正在評估是否轉職、轉產業或轉角色 |
| **核心需求** | 評估轉職時機是否成熟；了解目標角色/產業的市場現況；識別自己技能的可轉移方向 |
| **痛點** | 轉職決定影響重大，缺乏外部客觀數據；身邊的人都給出主觀建議；不知道轉型方向是否符合市場趨勢 |
| **使用場景** | 考慮辭職前做市場驗證；選擇進修方向前確認市場需求；評估目標公司所在產業的健康度 |
| **行動觸發點** | 看到目前角色的 AI 衝擊評級升高，同時看到目標轉職角色的職缺量在擴張 |

### 次要受眾

#### 受眾 C：職涯顧問 / 就業輔導員

| 面向 | 說明 |
|------|------|
| **人口特徵** | 獨立職涯顧問、大學就業輔導中心老師、政府就業服務站人員 |
| **核心需求** | 有數據支撐的市場現況資料，強化諮詢的客觀性；快速掌握本週就業市場動態 |
| **使用場景** | 諮詢前的市場資料準備；輔導學生/客戶時引用具體數據 |

#### 受眾 D：應屆畢業生 / 職場新鮮人

| 面向 | 說明 |
|------|------|
| **人口特徵** | 22-25 歲，即將或剛進入職場，對就業市場認知尚淺 |
| **核心需求** | 了解各角色的入門要求和市場需求；建立對就業市場的基本認知框架 |
| **使用場景** | 確認畢業後的求職方向；了解哪些技能有助於起薪 |
| **注意**：需要更多基礎解釋，避免過度假設先備知識 |

## 3. 差異化定位

### 相對競爭者的差異

| 競爭者類型 | 代表 | 我們的差異 |
|------------|------|------------|
| **職涯諮詢服務** | 個人職涯顧問、104 VIP 諮詢、履歷健診服務 | 數據基底 vs 顧問經驗主觀性；免費 vs 付費（千至數萬元）；市場層面 vs 個人層面 |
| **商業職涯媒體** | 換日線、CakeResume Blog、商周職場版 | 週頻數據驅動 vs 不定期編輯選題；市場觀測整合 vs 個別故事/意見分享 |
| **職涯書籍** | 各類職涯發展書籍 | 即時本週市場數據 vs 靜態通用原則；台灣市場具體性 vs 泛用適用性 |
| **AI 職涯助理** | 各類 AI 履歷優化工具 | 基於真實市場觀測數據 vs 模型訓練的通用建議；每週更新 vs 固定知識截點 |
| **求職論壇** | PTT Soft_Job、Dcard 職場板 | 系統性分析 vs 個人案例分享；數據支撐 vs 個人經驗主觀性 |

### 核心獨特優勢

1. **整合所有 4 個前序報告**：climate_index + skills_drift + industry_segments + salary_bands 的綜合結論，是系統最終的行動輸出
2. **本週市場數據驅動**：不是上季或去年的統計，就是本週觀測的職缺、技能、薪資數據
3. **AI 取代向量職涯風險評估**：53 個角色的取代風險評估，幫助讀者在職涯規劃中納入 AI 因素
4. **明確的「我們不做什麼」邊界**：承認 AI 系統的局限性，不假裝能替代個人化諮詢，建立誠實的信任關係
5. **免費公開**：相對於昂貴的職涯諮詢，降低了使用門檻

### 我們不做的事

- 不給出「你應該做 X」的確定性個人建議
- 不預測特定時間範圍的職業消亡（如「3年內此職業將消失」）
- 不推薦特定付費課程、培訓機構或服務
- 不對個人情況（年齡、學歷、家庭背景）做評判或建議
- 不提供法律或財務建議（如是否申請失業給付）
- 不取代個人化的職涯諮詢——我們提供市場數據層面的參考，不是個人決策

## 4. 品牌調性

### 語氣 (Tone of Voice)

| 維度 | 描述 |
|------|------|
| **整體調性** | 有數據的好朋友，而非權威專家。「這週數據看起來是這樣的，你可以考慮……」而非「你應該……」 |
| **對求職者說話** | 賦權而非指令；承認複雜性（每個人情況不同）；具體可行動（不是泛泛而談） |
| **對轉職者說話** | 市場現實感（數據說什麼就說什麼）+ 可能性感（看到數據中的機會窗口） |
| **對職涯顧問說話** | 提供客觀市場資料，不挑戰諮詢師的專業判斷 |
| **語言節制** | 使用「數據顯示」「市場觀測到」「值得參考」，避免「應該」「必須」「一定要」 |
| **應避免** | 恐嚇式的 AI 取代論（「你的工作快消失了！」）；過度樂觀的機會渲染；確定性的個人建議 |

### 視覺語言

| 元素 | 風格 |
|------|------|
| **受眾分流設計** | 報告開頭明確分為「求職中」「考慮轉職」「在職觀察」三條閱讀路徑 |
| **行動清單格式** | 「本週可以做的事」用 checkbox 格式呈現，但強調「參考方向」而非「待辦事項」 |
| **機會窗口視覺化** | 本週適合投遞的角色/產業用綠燈/黃燈/紅燈系統標示 |
| **免責聲明設計** | 醒目但不突兀，整合在報告設計中而非埋在頁尾小字 |

### 關鍵詞

- 求職策略、轉職規劃、職涯路徑
- 本週求職建議、就業市場機會
- AI 取代職業、技能轉型、職涯安全性
- 台灣就業市場建議、數據驅動職涯規劃
- 哪些技能現在最值錢、哪個產業最缺人

## 6. 改版目標

### P0（阻擋發布的問題）

- [ ] 免責聲明必須在報告開頭清晰呈現，不可埋在頁尾（本 Mode 為最高風險 Mode）
- [ ] 所有建議語句必須使用非確定性語氣（「值得考慮」「數據顯示」而非「應該」「必須」）
- [ ] 明確標示本報告依賴前序 Mode 數據，若某 Mode 未執行，需說明對分析深度的影響
- [ ] 不得推薦任何具體的付費課程、培訓機構或服務（即使匿名提及也不行）

- [ ] 增加「受眾分流」：報告開頭提供「求職中」「考慮轉職」「在職觀察」三條閱讀路徑，讓不同狀態的讀者快速找到最相關的部分
