## 1. 受眾深度分析

### 1.1 主要受眾 A：積極求職中的工作者

#### Persona 摘要

| 項目 | 描述 |
|------|------|
| 角色 | 已開始投遞或即將開始的求職者 |
| 年齡/背景 | 25-40 歲，希望提高求職成效 |
| 目標 | 知道本週哪些角色和產業的機會較多，得到具體行動方向 |
| 痛點 | 求職建議通常泛用、不知道「現在」的市場具體狀況 |
| 行為特徵 | 每週一固定瀏覽，最重視「本週具體可以做什麼」 |

#### Jobs-to-be-Done

| 任務類型 | 具體任務 |
|----------|----------|
| 功能性 | 掌握本週最值得投遞的角色和產業組合 |
| 功能性 | 知道該補什麼技能才能提高面試機率 |
| 功能性 | 評估自己目標角色的 AI 取代風險 |
| 情感性 | 在迷茫的求職過程中獲得方向感——「至少我知道這週該做什麼」 |
| 情感性 | 用數據支撐的行動計畫取代焦慮和無助感 |
| 社會性 | 在面試時展現對市場的了解，增加錄取機率 |

#### 使用者旅程

| 階段 | 行為 | 問題/疑慮 | 需要的資訊 | 期望的體驗 |
|------|------|-----------|------------|------------|
| 意識 | 搜尋「現在該找什麼工作」「2026 求職策略」 | 有沒有人用數據告訴我現在該怎麼做？ | SEO 含「求職策略」「轉職」 | 搜尋結果看起來是數據驅動的 |
| 考慮 | 看免責聲明和市場概覽 | 這個建議可靠嗎？AI 產出的能信嗎？ | 免責聲明、數據來源、依賴 Mode | 免責聲明誠實但不嚇人 |
| 決策 | 找到與自己角色相關的建議 | 對我的職位有什麼具體建議？ | 按角色/向量分類的建議 | 快速跳轉到自己的類別 |
| 使用 | 執行行動清單 | 這些行動真的有效嗎？ | 行動與數據的連結說明 | 可打勾的具體待辦清單 |
| 推薦 | 分享到求職社群 | 這個報告品質穩定嗎？ | 連續更新紀錄 | 社群分享友好 |

#### 資訊需求

| 想知道什麼 | 擔心什麼 | 需要的證據 |
|------------|----------|------------|
| 本週哪些角色職缺最多？ | 按照 AI 建議做會不會走錯方向 | 明確的數據來源引用 |
| 我的技能還有市場嗎？ | 自己的職業會不會被 AI 取代 | AI 向量評估有具體數據支撐 |
| 補什麼技能最有效？ | 學了也找不到工作 | 技能需求上升的具體數據 + 職缺數 |

| 問題類型 | 具體問題 | 根因 | 影響範圍 | 解決方向 |

| 內容不相關 | 偏向科技職缺建議 | 資料源以科技職缺為主（HN Hiring, Arbeitnow） | 非科技職業的台灣讀者 | 深化 tw_govjobs 的非科技職缺分析 |

## 4. 機會識別

### 4.1 必須補的內容

| 內容 | 為什麼必須 | 競品參考 |
|------|------------|----------|
| 受眾分流設計 | 報告最長、受眾最多元，缺乏導引會流失讀者 | 104 職場力按受眾分類文章 |

### 4.2 差異化機會

| 機會 | 為什麼是機會 | 如何執行 |
|------|--------------|----------|
| 數據驅動的每週行動清單 | 競品給的是泛用建議，本站基於本週數據 | 強化行動清單與數據的連結 |
| AI 取代向量職涯風險評估 | 業界無人以五向量框架系統評估職涯風險 | 加入視覺化風險矩陣 |
| 跨 Mode 整合建議 | 競品不做市場溫度+技能+產業+薪資的整合分析 | 維持並強化四 Mode 整合能力 |
| 免責聲明即品牌資產 | AI 生成建議的最大質疑是可靠性，透明免責反而建立信任 | 將免責聲明設計為「誠實的 AI」品牌信號 |

### 4.3 強化重點

| 現有優勢 | 如何強化 | 預期效果 |
|----------|----------|----------|
| 免責聲明最完備 | 維持業界最高標準 | 持續法律風險控管 + 品牌信任 |
| 跨 Mode 整合 | 明確標註每段建議來自哪個 Mode | 強化數據可追溯性 |
| 多受眾行動清單 | 擴充台灣非科技職業建議 | 覆蓋更多讀者 |

## 5. 關鍵發現摘要

### 最重要的 3 個發現

1. **受眾分流是最迫切的體驗改善**：career_strategy 是系統中最長、受眾最多元的報告。求職中、考慮轉職、在職觀察的讀者需求截然不同，但目前被迫線性閱讀全文。加入開頭分流設計可大幅提升每個受眾的相關性感受。

2. **台灣非科技職業覆蓋不足是受眾廣度瓶頸**：目前建議偏向科技職缺（資料源限制），但台灣求職市場中業務、行政、製造等職業佔比極高。加入這些職業的專屬建議段落可觸及更大的潛在讀者群。

3. **免責聲明是品牌差異化資產而非負擔**：在 AI 生成內容信任度普遍偏低的環境下，career_strategy 的完備免責聲明反而成為「我們是誠實的 AI」的品牌信號。應將免責聲明設計為品牌的一部分，而非被動的法律防護。

### 改版重點方向

1. **受眾分流設計**：報告開頭按「求職中」「考慮轉職」「在職觀察」三條路徑分流
2. **台灣在地化**：為常見非科技職業加入專屬建議段落
3. **AI 風險視覺化**：五向量風險矩陣圖，讓讀者快速定位自己的風險等級
